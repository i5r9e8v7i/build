Booking System Development Brief

Platform: WordPress (self-hosted)

Stack: Bookly Pro + Add-ons + WPML or TranslatePress

Timeline: 7â€“10 business days

Core Functional Requirements

  • Hundreds of bookable services (each with its own name, duration, price, and assigned Google Calendar)
  • Real-time availability pulled from corresponding Google Calendars (100+ total)
  • Each booking creates a calendar event and prevents double-booking
  • Booking without login (guest checkout)
  • Multi-service booking (cart functionality) with full or partial payment per service
  • Email notifications sent to the relevant service provider and customer
  • Admin should be able to manage services (create/edit/delete), with each service linked to its own Google Calendar and notification email
  • Admin dashboard with filters, stats, CSV/PDF export

Payment Integration

  • Support for one of the following gateways: Square, Stripe, Adyen, or Mollie
  • Multi-currency support (USD, EUR, GBP, JPY, KRW, TWD etc.)
  • Currency pre-selected by browser language/location; manual switch optional

Language Support

  • Frontend supports at least 4-6 languages (e.g. English, Spanish, French, Japanese, Korean, Chinese)
  • Booking forms, service descriptions, and notifications must be translatable
  • Confirmation emails sent in the language selected at booking
  • Frontend language switcher required

Tech Stack Notes

  • Use WPML or TranslatePress for multilingual setup
  • Use WooCommerce if needed for advanced cart or payment logic
  • Developer should handle plugin setup, SSL, calendar API integration, and notification rules

What Client Will Provide

  • Service catalog with details and calendar assignment
  • Domain, hosting, and translation files

Deliverables

  • Fully functional multilingual booking site (WordPress)
  • Each service linked to its own Google Calendar
  • Real-time availability & automatic event creation
  • Frontend booking form with multi-service checkout
  • Email logic and admin dashboard
  • Payment setup (1 selected gateway)
  • Multi-language support with email localization

Note: This is not a plugin development project. The solution should be built using existing tools with expert-level configuration, and must not rely on any third-party services requiring recurring subscription fees.
